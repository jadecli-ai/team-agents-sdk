# Creating and Editing Milestones for Issues and Pull Requests

> Source: https://docs.github.com/en/issues/using-labels-and-milestones-to-track-work/creating-and-editing-milestones-for-issues-and-pull-requests

## Overview

You can create a milestone to track progress on groups of issues or pull requests in a repository.

## Creating and Editing Milestones

### Step 1: Navigate to Your Repository

On GitHub, go to the main page of your repository.

### Step 2: Access Issues or Pull Requests

Under your repository name, select either **Issues** or **Pull Requests** from the navigation tabs.

### Step 3: Open Milestones

Next to the search field, click the **Milestones** button (marked with a signpost icon).

### Step 4: Choose Your Action

You have two options:

- **Create a New Milestone**: Click the **New Milestone** button
- **Edit an Existing Milestone**: Click **Edit** next to the milestone you want to modify

### Step 5: Configure Your Milestone

Enter the milestone's title, description, or make other changes as needed. Then click either **Create milestone** or **Save changes** to confirm.

### Markdown Support

Milestones support Markdown syntax rendering. For formatting guidance, consult the documentation on basic writing and formatting syntax.

## Deleting Milestones

When you delete milestones, issues and pull requests are not affected.

### Deletion Steps

1. Navigate to your repository's main page
2. Click **Issues** or **Pull Requests**
3. Click the **Milestones** button next to the search field
4. Locate the milestone you want to remove
5. Click the red **Delete** link beside it

The milestone will be removed without impacting associated issues or pull requests.
