# Creating a Project

> Source: https://docs.github.com/en/issues/planning-and-tracking-with-projects/creating-projects/creating-a-project

## Overview

Projects are an adaptable collection of items that stay up-to-date with GitHub data. They enable tracking of issues, pull requests, and notes with custom fields and purpose-built views. Organizations and users can create blank projects or leverage existing templates.

## Creating a Project

### Organization Projects

Organization-level projects can monitor issues and pull requests across repositories within the organization. Templates can be established and curated for team members.

**Steps:**

1. Click your profile picture (upper-right corner) then select **Organizations**
2. Choose your organization name
3. Navigate to **Projects** tab beneath organization name
4. Click **New project**
5. Select project type:
   - Blank projects: Choose **Table**, **Roadmap**, or **Board**
   - Template-based: Select from built-in, organization-created, or recommended templates
6. Review template fields, views, workflows, and insights (if applicable)
7. Enter project name in text field
8. Optionally select "Import items from repository" to populate with existing/new items
9. Click **Create project**

### User Projects

Personal user projects track issues and pull requests from repositories owned by individual accounts.

**Steps:**

1. Click profile picture (top-right) then **Your profile**
2. Navigate to **Projects** tab on profile
3. Click **New project**
4. Select project type:
   - Blank: Choose **Table**, **Roadmap**, or **Board**
   - Template: Select built-in template
5. Review template details if selected
6. Enter project name
7. Click **Create project**

## Updating Project Description and README

Descriptions and READMEs communicate project purpose, usage instructions, and relevant links.

**Steps:**

1. Navigate to your project
2. Click menu icon (top-right)
3. Select **Settings**
4. Add description under "Add a description" then type and click **Save**
5. Update README under "README" section:
   - Use Markdown formatting
   - Toggle between text and preview
6. Click **Save**

Access quick edits via sidebar expansion icon without entering full settings.
